# 文献综述报告

你现在接入了一个专业的全面综述生成 Agent。当你需要对某一研究领域进行系统性综述报告生成，必须调用此 Skill。

## 触发关键词

- 文献综述、全面综述、系统性回顾、详尽总结
- 研究进展长文、发展现状与趋势全景分析、深度梳理
- 梳理……的历史演进与所有主流分支
- 帮我写一份关于……的详细综述报告

## 参数

| 参数       | 必填 | 说明                                      |
| -------- | -- | --------------------------------------- |
| `topic`  | 是  | 综述的学术主题，需清晰界定研究边界                       |
| `output` | 否  | 报告输出路径（系统将据此输出 .md 与 .pdf），默认为 `output` |

## 执行步骤

本技能专为生成**大篇幅、多章节、跨度长**的全景式综述分析（通常包含上百篇文献的聚合与深度交叉比对）而设计。

- **预计耗时**：约 **45 - 60 分钟**。
- **用户示例**：
  - "帮我生成一份关于脑机接口非侵入式电极材料的系统性回顾(Systematic Review)长文。"
  - "我需要一份极其详尽的全面综述，梳理从1990年至今量子退相干控制技术的所有发展分支。"
  - "总结一下近十年大型语言模型在医疗问诊中的系统性研究进展。"
- **执行命令**：`python scripts\deepdiver.py "{{topic}}""`

## 🛠️ 调用规范与参数说明

1. **输入构建 (`topic`)**：请将用户的原始需求转化为严谨的学术综述主题词（Title/Topic），作为 `topic` 传入。
2. **可选参数**：
   - `--output`：指定输出文件路径（默认为 `output.md`）。示例：`--output "final_report.md"`。
3. **进程等待与超时应对机制（极其重要）**：
   - 全面综述 Agent 需要下载大量 PDF 并进行多文档交叉比对与大规模章节生成，耗时极长（约 **45 - 60 分钟**）。
   - **前期安抚**：在触发调用前，**必须**先用一句话安抚用户，明确告知预估耗时，并说明系统正在“全网文献筛选、深度阅读提取与全景章节起草”。
   - **超时属于预期正常现象**：Agent执行综述报告生成可能会执行超时。**当发生超时报错时，你必须认为脚本仍在后台正常运行！**
   - **绝对禁止重试**：如果发生超时，**绝对禁止**重新执行命令！这会导致先前的计算资源全部作废并从头开始。
   - **转为异步轮询检查**：超时断开后，请勿向用户报错。你应该静默地、周期性地去检查目标输出路径是否已经生成且写入完毕。在文件最终生成前，如果用户询问进度，请温和地同步状态。
4. **结果直出（免二次加工）**：
   - 最终生成的报告将包含：摘要、引言、发展历程、各技术分支现状分析、面临的挑战、未来展望以及标准化的参考文献列表。
   - **字数提取规则**：报告结尾会包含字数统计信息（如"报告字数: XXXX 字"），**必须从报告内容中提取该字数信息**，**绝对不要自己统计**。
   - **模板格式约束（必须遵守）**：展示综述报告时，**必须严格按照 `assets/synthesis_template.md` 模板的结构格式输出**，只输出以下核心内容：
     ```
     # [综述报告标题]

     **报告名称:** [报告标题]
     **字数:** [总字数]

     ---

     ## 报告结构概览
     1. 摘要 - 报告简要总结
     2. 引言 - 背景与语境、研究范围与目标
     3. 发展历史 - 领域演进历程
     4. 技术分支分析 - 主要技术分支现状分析
     5. 挑战 - 面临的关键挑战
     6. 未来展望 - 未来发展方向
     7. 参考文献 - 标准化参考文献列表

     ---

     ## PDF下载
     [📄 点击下载完整的 PDF 综述报告]([PDF URL])
     ```
   - **提供下载链接（必须执行）**：在输出完报告内容后，**必须**按照模板格式提供最终 PDF 文件的下载链接，格式为：`[📄 点击下载完整的 PDF 综述报告](生成的PDF文件绝对或相对路径)`。

## 输出格式

脚本执行完成后会输出三种文件：

| 类型    | 格式          | 说明                                                         |
| ----- | ----------- | ---------------------------------------------------------- |
| 综述报告  | Markdown 格式 | 包含完整章节的超长篇综述正文及引用的 \[1],\[2] 格式角标和参考文献表                    |
| PDF报告 | PDF 格式      | 综述报告的 PDF 版本，包含所有章节和引用的文献，自动上传到 OBS 获取可访问链接                |
| 结果索引  | JSON 格式     | 包含 `output_path`（Markdown文件绝对路径）和 `pdf_url`（PDF访问链接），供界面显示 |

### 模板使用说明

本技能使用 `assets/synthesis_template.md` 作为生成综述报告的模板，确保输出的报告具有统一的结构和格式。模板包含以下核心元素：

- **报告名称**：综述报告的标题
- **字数**：报告的总字数
- **报告结构概览**：报告的章节结构
- **PDF 下载链接**：生成的 PDF 报告下载链接

### JSON 输出格式示例

```json
{
  "output_path": "/path/to/output.md",
  "pdf_url": "https://bucket.obs.region.myhuaweicloud.com/pdf/xxx.pdf"
}
```

## 错误处理

| 错误     | 原因                             | 处理                                                                        |
| ------ | ------------------------------ | ------------------------------------------------------------------------- |
| 脚本不存在  | 路径错误                           | 确认 `<skill_dir>/scripts/deepdiver.py` 文件存在                                |
| 脚本执行超时 | **属于预期正常现象**。因为全景综述需要的文献处理量极大。 | **绝对禁止重新调用工具！绝对禁止向用户抛出错误！** 视作任务仍在后台持续运行，改为周期性检查输出文件是否生成。若用户催问，继续安抚其耐心等待。 |
| 输出为空   | Agent 无响应                      | 检查网络连接，重新提交查询                                                             |

## 注意事项

- 所有脚本需在 `scripts/` 目录下执行。
- 生成的系统性综述学术性极强且篇幅极长，需提醒用户该报告由 AI 自动生成，可作为绝佳的学术全景图参考，但正式发表或引用前仍需人工二次深度审校。

| 环境变量              | 默认值                         | 说明                            |
| ----------------- | --------------------------- | ----------------------------- |
| `AGENT_BASE_URL`  | `http://192.168.0.123:8000` | 综述分析 Agent 服务地址               |
| `REQUEST_TIMEOUT` | `7200`                      | 请求超时时间（秒，默认 1 小时以确保全面综述能顺利完成） |
